# Gem Smash Notes

This file documents the current `GemSmash.html` game so future iterations have a shared reference point. Add new decisions and changes here as the game evolves.

## Files

- `GemSmash.html`: Single-file offline mobile match-three game with embedded HTML, CSS, and JavaScript.
- `GemSmash.md`: Living project notes, feature outline, and change history.

## Game Overview

Gem Smash is a portrait-first match-three puzzle game built as one standalone HTML file. It uses an 8x8 board, colored gem SVGs, level objectives, special gems, boosters, save data, and mobile-friendly tap/swipe controls.

The game is designed to run locally in a browser with no build step and no external assets.

## Core Gameplay

- Match 3 or more gems by swapping adjacent gems.
- Valid swaps consume one move.
- Invalid swaps are reverted and do not consume a move.
- Cascades resolve automatically after matches clear.
- The board is reshuffled when no moves are available.
- Scores increase based on cleared gem count and cascade chain depth.

## Level System

- `MAX_LEVELS` is currently `200`.
- Levels are generated from `levelConfig(level)`.
- Level config controls:
  - Available gem palette size.
  - Move count.
  - Frost tile count.
  - Score target.
  - Collection targets.
  - Objective type.
- The difficulty curve now uses budget caps so generated objectives stay within the current move count and gem palette size.
- Palette growth is staged more slowly:
  - Levels 1-20 use 4 gem colors.
  - Levels 21-40 use 5 gem colors.
  - Levels 41+ use 6 gem colors.
- Base moves now start at 25 and decline gradually to a floor of 18.
- Collection, frost, and score targets each use a raw growth curve plus a move-budget cap so upper levels remain challenging without asking for unrealistic clears.
- Progress is saved to `localStorage` under `lynettes-smash-save-v1`.

## Objective Types

The level generator currently rotates through these objective types:

- `score`: reach a target score.
- `collect`: collect a target number of one gem type.
- `frost`: clear a target number of frosted tiles.
- `scoreCollect`: reach a target score and collect a gem target.
- `doubleCollect`: collect two gem targets.

## Special Gems

Special gems are created from larger matches:

- 4-gem row match creates a row clearer.
- 4-gem column match creates a column clearer.
- 5+ gem match creates a burst clearer.

Special gem effects:

- Row special clears the whole row.
- Column special clears the whole column.
- Burst special clears a 3x3 area.

Special clears flash before resolving and contribute to score, collection, frost clearing, biggest smash, and chain stats.

## Boosters

The booster types are tracked with `BOOSTER_TYPES`:

- `hammer`: clears one selected gem/tile.
- `shuffle`: reshuffles the board.
- `line`: Blast booster that clears the selected row and column.

Boosters shown in the UI are the combined total of:

- Earned inventory, which persists across levels.
- Retry assist boosters, which are temporary helpers for the current stuck level.

When a booster is used, retry-assist boosters are spent before earned inventory.

## Retry Assist

Retry assist is controlled by `retryAssist(level)` and is based on the number of failed attempts saved for that level.

Current behavior:

- 2-5 failures: `+3` extra moves.
- 6-8 failures: `+6` extra moves.
- 9-11 failures: `+9` extra moves.
- Additional failures continue adding moves in 3-move blocks.
- 4+ failures: grants a temporary Hammer.
- 6+ failures: grants a temporary Shuffle.
- 8+ failures: grants a temporary Blast.

The assist message is generated by `assistMessage(failures, extraMoves)` and appears when a retry starts or when the player fails a level.

## Earned Power-Ups

Players can now earn persistent boosters through level performance.

Current reward rules:

- Improving a level to 2 stars earns `1 Hammer`.
- Improving a level to 3 stars earns `1 Shuffle`.
- First clearing every 5th level earns an extra `Hammer`.
- First clearing every 10th level earns a `Blast`.

Rewards are calculated in `completionReward(level, previousStars, stars)`, applied through `addReward(reward)`, and summarized in the level-complete modal with `rewardText(reward)`.

Replaying a level only grants star rewards for newly earned stars. For example, improving a previous 1-star clear to 3 stars grants the 2-star and 3-star rewards once.

## Save Data

Saved state currently includes:

- Selected/current level.
- Highest unlocked level.
- Sound setting.
- Hint setting.
- Level completions.
- Level failure counts.
- Earned booster inventory.

Clear Data resets progress, failures, settings, earned inventory, temporary assist boosters, and current booster counts.

## UI And Controls

Primary UI areas:

- Top bar with level, moves, score, menu, sound, and restart.
- Objective panel with live progress chips and progress bar.
- Booster bar with Hammer, Shuffle, and Blast.
- 8x8 board.
- Previous/current/next level strip.
- Title/menu dialog with Play and Levels tabs.
- Level-complete/try-again modal.

Controls:

- Tap a gem, then tap an adjacent gem to swap.
- Swipe a gem toward an adjacent tile to swap.
- Tap a booster, then tap a target for Hammer or Blast.
- Shuffle activates immediately.

## Mobile Behavior

- The game is optimized for portrait mobile play.
- Short landscape screens show a portrait-only overlay.
- Pointer events are used when available.
- Touch event fallback is included for browsers without `PointerEvent`.
- Swipe movement prevents page scrolling once the gesture is clearly intentional.
